$self->give_shop_points( %options )

The options hash MUST contain the following keys:

amount

How many points to give the user. May be positive or negative, s use a negative number to deduct points from a user's balance. Will not allow a user's balance to go negative.

reason

A short description of why this transaction is happening. For example: 'purchase of cart 9883463'.

The options hash MAY contain these keys, as well:

admin

If this action was being done by an administrator, pass their userid or user object here. This helps us record when admins make things happen.

Example usage:

$self->give_shop_points( amount => 50, reason => 'purchased' );

This gives 50 points to the user as a routine purchase.

$self->give_shop_points( amount => -100, reason => 'refund', admin => $remote );

Admin processed refund, remove 100 points from the user's balance.

Returns a true value on success, undef on error.

$self->shop_points

Returns how many points this user currently has available for spending in the shop. For adjusting points on a user, please see <$self-give_shop_points>>.

Userpic-Related Functions

$u->activate_userpics

Sets/unsets userpics as inactive based on account caps.

$u->allpics_base

Return the base URL for the icons page.

$u->clear_userpic_kw_map

Clears the internally cached mapping of userpics to keywords for this user.

$u->expunge_userpic( $picid )

Expunges a userpic so that the system will no longer deliver this userpic.

$u->get_keyword_from_mapid( $mapid, %opts )

Returns the keyword for the given mapid or undef if the mapid doesn't exist.

Arguments:

mapid

Additional options:

redir_callback

Called if the mapping is redirected to another mapping with the following arguments

( $u, $old_mapid, $new_mapid )

$u->get_mapid_from_keyword( $kw, %opts )

Returns the mapid for a given keyword.

Arguments:

kw

The keyword.

Additional options:

create

Should a mapid be created if one does not exist.

Default: 0

$u->get_picid_from_keyword( $kw, $default )

Returns the picid for a given keyword.

kw

Keyword to look up.

default (optional)

Default: the users default userpic.

$u->get_picid_from_mapid( $mapid, %opts )

Returns the picid for a given mapid.

Arguments:

mapid

Additional options:

default

Default: the users default userpic.

redir_callback

Called if the mapping is redirected to another mapping with the following arguments

( $u, $old_mapid, $new_mapid )

$u->get_userpic_count

Return the number of userpics.

$u->get_userpic_info( $opts )

Given a user, gets their userpic information

Arguments:

opts

Hashref of options

Valid options:

load_comments
load_urls
load_descriptions

Returns a hashref with the following keys:

comment

Maps a picid to a comment. May not be present if load_comments was not specified.

description

Maps a picid to a description. May not be present if load_descriptions was not specified.

kw

Maps a keyword to a pic hashref.

kwmap

Maps a keyword to a mapid.

map_redir

Maps a mapid to a diffrent mapid.

mapid

Maps a mapid to a pic hashref.

mapkw

Maps a mapid to a keyword.

pic

Maps a picid to a pic hashref.

$u->get_userpic_kw_map

Gets a mapping from userpic ids to keywords for this User.

$u->mogfs_userpic_key( $pic )

Make a mogilefs key for the given pic for the user.

Arguments:

pic

Either the userpic hash or the picid of the userpic.

$u->resolve_mapid_redirects( $mapid, %opts )

Resolve any mapid redirect, guarding against any redirect loops.

Returns: new map id, or 0 if the mapping cannot be resolved.

Arguments:

mapid

Additional options:

redir_callback

Called if the mapping is redirected to another mapping with the following arguments

( $u, $old_mapid, $new_mapid )

$u->userpic

Returns LJ::Userpic for default userpic, if it exists.

$u->userpic_have_mapid

Returns true if the userpicmap keyword mappings have a mapid column ( dversion 9 or higher )

$u->userpic_quota

Returns the number of userpics the user can upload (base account type cap + bonus slots purchased)
