NAME

DW::Template - Template Toolkit helpers for Apache2.

SYNOPSIS

API

$class->template_string( $filename, $opts, $extra )

Render a template to a string.

$class->cached_template_string( $key, $filename, $subref, $opts, $extra )

Render a template to a string -- optionally fragment caching it. $subref returns the options for template_string.

fragment opts:

lock_failed - The text returned by this subref is returned if the lock is failed and the grace period is up.
expire - Number of seconds the fragment is valid for
grace_period - Number of seconds that an expired fragment could still be served if the lock is in place

$class->render_cached_template( $key, $filename, $subref, $extra )

Render a template inside the sitescheme or alone.

See render_template, except note that the opts hash is returned by subref if it's needed.

$extra can contain:

no_sitescheme == render alone
title / windowtitle / head / bodyopts / ... == text to get thrown in the section if inside sitescheme
lock_failed = subref for lock failed.
expire - Number of seconds the fragment is valid for
grace_period - Number of seconds that an expired fragment could still be served if the lock is in place

$class->render_template( $filename, $opts, $extra )

Render a template inside the sitescheme or alone.

$extra can contain:

no_sitescheme == render alone
title / windowtitle / head / bodyopts / ... == text to get thrown in the section if inside sitescheme

$class->render_template_misc( $filename, $opts, $extra )

Render a template inside the sitescheme or alone. This can also be safely called ( with some work on the other side ) from a BML context and still spit the content where required. ( Note, the "alone" bit will be ignored from BML contexts )

Can safely directly return this from either trans/Controller, internal journal page generation or (most) BML contexts.

$extra can contain:

no_sitescheme == render alone

This will be ignored for 'bml' scopes.

title / windowtitle / head / bodyopts / ... == text to get thrown in the section if inside sitescheme
scope = Scope, accepts nothing, 'bml', or 'journal'
scope_data = Depends on scope
bml Hashref of scalar-refs of where to throw the sections
journal $opts hashref passed into LJ::make_journal and beyond.

$class->render_string( $string, $extra )

Render a string inside the sitescheme or alone.

$extra can contain:

no_sitescheme == render alone

If you are just printing text or other data, do not call DW::Template->render_string and instead just $r->print and return $r->OK.

This is mostly for being used from render_template.

title / windowtitle / head / bodyopts / ... == text to get thrown in the section if inside sitescheme

$class->render_scheme( $sitescheme, $body, $sections )

Render the body and sections in a TT sitescheme

sitescheme

A DW::SiteScheme object
