2# Note: this is a very early version of a CSS cleaner. The plan is to eventually
3# make it a white-listing CSS cleaner (deny by default) with a nice
4# interface where you can build policy about what's allowed, like
5# HTML::Sanitize/::Scrub/etc, but for now this is almost a null cleaner,
6# just parsing and reserializing the CSS, removing two trivial ways to
7# inject javascript.
8#
9# The plan now is to integrate this interface into LiveJournal, then improve
10# this module over time.
11#
12# Note2: we tried 4 different CSS parsers for this module to use, and all 4 sucked.
13# so for now this module sucks, until we can find a suitable parser. for the
14# record, CSS::Tiny, CSS, and CSS::SAC all didn't work. and csstidy wasn't
15# incredibly hot either. CSS.pm's grammar was buggy, and CSS::SAC had the
16# best interface (SAC) but terrible parsing of selectors. we'll probably
17# have to write our own, based on the Mozilla CSS parsing code.
